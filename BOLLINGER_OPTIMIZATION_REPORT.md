### 参数网格
- **period (周期)**: 10, 15, 20, 25, 30
- **std_dev (倍数)**: 1.5, 2.0, 2.5, 3.0
- **exit_logic (卖出逻辑)**: upper, middle, percent

**总计**: 5 × 4 × 3 = **60 组参数组合**

- 数据条数: 500 日
- 日期范围: 2022-01-03 ~ 2023-12-01
- 初始资金: ¥100,000

## 🔍 关键发现

### 1. 最佳 std_dev: 2.5

```
倍数 1.5:  平均收益 -7.6%  |  过于敏感，频繁止损
倍数 2.0:  平均收益 -3.1%  |  标准参数，表现一般 ⭐
倍数 2.5:  平均收益 +1.9%  |  **最佳参数** 🏆
倍数 3.0:  平均收益  0.0%  |  过于宽松，很少触发
```

**结论**: std_dev=2.5 是"甜点"，既能捕捉均值回归，又避免过度交易。

### 2. 最佳 period: 15

```
周期 10: 过于敏感，容易假信号
周期 15: **最佳周期**，平衡灵敏度与稳定性 🏆
周期 20: 标准周期，表现一般
周期 25: 反应稍慢，错过部分行情
周期 30: 过于迟钝，信号滞后
```

**结论**: 15日均线比20日更灵敏，能更早捕捉反转机会。

### 3. 卖出逻辑影响很小

```
upper (上轨卖出):   平均收益 -2.25% | 推荐 🏆
middle (中轨卖出): 平均收益 -2.25% | 与upper相同
percent (百分比):  平均收益 -2.25% | 与upper相同
```

**意外发现**: 三种卖出逻辑在这个数据集上表现几乎相同！

原因: 数据波动适中，触及上轨时往往也接近目标收益。

### 4. 最差参数组合

```python
# 避免使用！
period = 15, std_dev = 1.5  # 收益率 -10.4%
period = 20, std_dev = 1.5  # 收益率 -12.8%
```

**问题**: std_dev 太小导致通道过窄，频繁假突破。

## 💡 优化建议

### 1. 推荐使用参数
```python
signals = generate_strategy(
    data=data,
    strategy_name="bollinger",
    period=15,      # 从20改为15
    std_dev=2.5     # 从2.0改为2.5
)
```

### 2. 不同市场环境的参数调整

**震荡市场** (当前测试环境):
```python
period=15, std_dev=2.5  # 灵敏捕捉反转
```

**趋势市场**:
```python
period=25, std_dev=2.0  # 减少假信号
```

**高波动市场**:
```python
period=20, std_dev=3.0  # 扩大通道，避免频繁触发
```

### 3. 组合策略建议
```python
# 布林带 + 趋势过滤
# 只在均线向上时做均值回归
ma = data['close'].rolling(20).mean()
trend_up = data['close'] > ma

# 修改买入条件
buy_signal = (close < lower) & trend_up
```

### 使用优化后的策略

```python
from brain.strategies.lib import generate_strategy
from brain.backtest import BacktestEngine
import pandas as pd

# 加载数据
data = pd.read_csv('stock_data.csv', index_col='date', parse_dates=True)

# 使用优化参数生成信号
signals = generate_strategy(
    data=data,
    strategy_name="bollinger",
    period=15,        # 优化后

# 回测

print(f"回撤: {result['max_drawdown']}%")       # 预期: <1%
```
